API

About this API


The API for this website will allow clients to send json data specifying the location they are in, the time of day and what type of TV is being used. In response the API will return the time of peak data traffic for that day.

Additionally, clients can also send the brands and model numbers of TVs to this API and in response, the API will send the energy class of the TV from A to G with A representing energy efficient and G representing energy inefficiency.

If you are not within one of these areas, you can send a request to the API which will return the time of average peak data traffic amongst all of these regions. If your users are not using a TV, please specify the attributes model and brand within the JSON request as 'null'.

How to use the API


To use this API, please send JSON post request to the url: http://localhost:8000/api/

Within this post request you need to write the attributes location, day, brand and model like so:

{ "location": "outer london", "day": "saturday", "brand": "samsung", "model": "HG50EJ690YA" }

To specify the location please use (location JSON attributes are case insensitive):

North America
London
Nairobi
Manchester
Scotland
Wales
Middle East

To specify the day please use any day of the week (do not input dates, inputs are also case insensitive): To specify the brand please use the brand name. Examples of brands that would be recognized include samsung, panasonic, TCL. Please note that brand names are case insensitive. Please ensure that the model number is as accurate as possible as the model attribute of the JSON request is case sensitive. Alternatively, if the device is not a TV, please specify the TV model and brand within the JSON request as 'null'. Another attribute for the API request which can be used is the tv_agent_string JSON attribute where users can place the tv user agent string inside.

In response, the api will return a peak value which will represent the average peak data traffic time for that day. The API will also return the overall energy class, the energy class in standard dynamic range and the energy class in high dynamic range if a TV is both included within the JSON request and within the site's TV database and in response the api will also return energy data corresponding to the user agent string if a tv user agent string is present within the API. Here is an example JSON request:

{ "location": "middle east", "day": "saturday", "model": "T1931SAW-B5 B", "brand": "IIYAMA", "tv_agent_string": "Mozilla/5.0 (SMART-TV; Linux; Tizen 6.0) AppleWebKit/537.36 (KHTML, like Gecko) 4.0 Chrome/76.0.3809.146 TV Safari/537.36" }

and here is an example API response:

{ "peak": "20:00:00 UTC", "energy data": { "energy data": [ { "energy_class": "E", "energy_class_sdr": "E", "energy_class_hdr": "NA" } ] }, "agent_string_data": { "agent-string_data": [ { "energy_class": "G", "energy_class_sdr": "G", "energy_class_hdr": "G" } ] } }

Invalid Responses and no corresponding TVs Response


The times will always be in UTC so depending on your location, please handle the conversion in time appropriately. Additionally, if your API request does not conform to the location and day attributes above by either being an invalid location orday or being invalid in JSON structure, you will receive a specific error response detailing your error like so:

{ "error": "Invalid day" }

{ "error": "Invalid location" }

{ "error": "Invalid JSON in request body" }

If the day and location are valid and the TV sent does not correspond with the TVs stored within the site's database, the API will respond like so:

{ "peak": "09:00:00 UTC", "energy data": { "energy data": "No TV corresponds with TV sent in API request" } }

Here are some valid JSON request examples


{ "location": "outer london", "day": "monday", "model": "HG75EJ690YA", "brand": "samsung" "tv_agent_string": "Mozilla/5.0 (SMART-TV; Linux; Samsung 6.0) Model/HG50EJ690YA 4.0 Chrome/76.0.3809.146 TV Safari/537.36" }

{ "location": "middle east", "day": "Friday", "model": "null", "brand": "null" "tv_agent_string": "null" }
